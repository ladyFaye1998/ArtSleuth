Methodology

How ArtSleuth bridges art-historical connoisseurship and computational analysis

The Morellian Tradition

In the 1870s, Giovanni Morelli proposed that attribution should focus on incidental details — the shape of fingernails, the curl of earlobes, the rendering of drapery folds. These peripheral passages, executed habitually and without deliberation, reveal the artist's hand more reliably than any consciously composed focal passage. Bernard Berenson refined this into a systematic method of connoisseurship, and subsequent scholars extended it to address workshop production.

ArtSleuth formalises the Morellian intuition as a feature-extraction problem: the "incidental details" that Morelli identified correspond to the low-level textural and structural features that self-supervised vision transformers encode.

Dual Backbone Design

DINOv2 — Texture & Structure

DINOv2 (Oquab et al., 2024) learns visual representations without labelled data via self-distillation. The resulting feature space encodes texture, directionality, and granularity — the physical surface qualities that connoisseurs evaluate. This is the "hand of the artist" backbone.

CLIP — Semantic Context

CLIP (Radford et al., 2021) encodes images and text in a shared embedding space. This is critical for style classification, where categories like "Baroque" or "Impressionism" are culturally constructed labels — not purely visual properties. CLIP's linguistic grounding captures the semantic associations that define these categories.

Pipeline Stages

1. Preprocessing

Standard vision-transformer preprocessing: resize, centre-crop, and ImageNet normalisation. Three optional art-specific corrective transforms are available but not applied on the default inference path:

  • Varnish correction — attenuates warm amber shift from aged surface coatings
  • Craquelure suppression — selective median filtering for age-induced crack patterns
  • Canvas texture normalisation — frequency-domain filtering for periodic weave patterns

2. Patch Extraction

The painting is divided into local patches using one of three strategies: Grid (uniform tiling), Salient (high-gradient-energy regions), or Adaptive (default: grid + saliency-weighted oversampling).

3. Brushstroke Analysis

For each patch, the 2×2 structure tensor is computed from the greyscale gradient field. The eigenvalues yield orientation, coherence, and energy. Patch embeddings from DINOv2 are clustered (k-means) to reveal regions of homogeneous brushwork.

J = [ ⟨gx²⟩ ⟨gxgy⟩ / ⟨gxgy⟩ ⟨gy²⟩ ]
Coherence = (λ1 − λ2) / (λ1 + λ2)  Energy = √(λ1 + λ2)

4. Style Classification

The full image is encoded via CLIP, and the embedding is projected through three independent linear heads (period, school, genre), producing calibrated probability distributions over the respective taxonomies.

5. Attribution

The query painting's fused embedding (CLIP style + mean DINOv2 patch) is compared against a reference gallery using temperature-scaled cosine similarity. Confidence intervals are derived via the normal approximation to the binomial proportion.

6. Forgery Detection

A one-class anomaly detector evaluates the Mahalanobis distance of the query embedding from the reference distribution, normalised to a 0–1 anomaly score via sigmoid transform. Per-dimension z-scores identify which embedding dimensions contribute most.

DM(x) = √( (x − μ)T Σ−1 (x − μ) )

7. Cross-Attention Fusion (training-time)

For benchmark fine-tuning, a style-guided cross-attention module lets CLIP embeddings (queries) attend over DINOv2 patch tokens (keys/values). Multi-head attention (8 heads, 512 internal dim) with learned temperature. This is used during end-to-end training but is not part of the default inference path.

8. Temporal Style Drift

For artists with time-stamped reference works: PCA-reduce embeddings (up to 20 components), fit a GP with RBF + White noise kernel over normalised years, search the artist's active period for the year where the GP-predicted embedding is closest to the query, and compute a plausibility score with 95% confidence band.

9. Workshop Decomposition

Rather than flat k-means (fixed cluster count), ArtSleuth uses a variational Bayesian Gaussian mixture model with Dirichlet process prior, allowing the model to infer the number of hands. Components capturing fewer than 5% of patches are pruned. Each assignment carries a posterior probability, enabling soft boundaries.

10. Adversarial Robustness

A robustness testing framework simulates four forgery techniques: artificial aging (varnish + craquelure), style transfer perturbation, material anachronism injection, and composite forgery. The evaluator sweeps severity levels and reports per-technique detection rates.

Limitations

  • Training data bias: Pre-trained backbones encode biases from predominantly Western photographic training data.
  • Probabilistic, not definitive: All outputs are statistical estimates informing expert judgement.
  • Reference corpus dependency: Attribution and forgery detection quality scales with reference gallery size.

Get Started

Installation & Usage

From pip install to first analysis in under a minute

Command Line

# Install
pip install artsleuth

# Quick analysis
artsleuth analyze painting.jpg

# With web UI
pip install artsleuth[web,benchmarks]
artsleuth demo

# Style classification only
artsleuth style painting.jpg

# Compare two works
artsleuth compare a.jpg b.jpg

Python API

# Full pipeline
from artsleuth import analyze

result = analyze("painting.jpg")
print(result.summary())

# Style classification
from artsleuth import StyleClassifier
clf = StyleClassifier()
report = clf.classify(image)

# Forgery screening
from artsleuth import ForgeryDetector
det = ForgeryDetector()
result = det.detect(image,
  reference_artist="Vermeer")

MCP Server (AI Agent Integration)

# Start the Model Context Protocol server
artsleuth mcp-server

# In your MCP config (.cursor/mcp.json):
{
  "mcpServers": {
    "artsleuth": {
      "command": "artsleuth",
      "args": ["mcp-server"]
    }
  }
}
